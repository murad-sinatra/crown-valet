## MVP Outcome

The first shippable version should let a pilot venue run a complete valet shift:

- Staff checks in vehicles.
- Customers receive a digital ticket.
- Staff records vehicle status and parking location.
- Customers request pickup from the ticket page.
- Staff processes the pickup queue.
- Managers monitor active vehicles and basic daily performance.
- Every session keeps a traceable status timeline.

## MVP Principle

The MVP should prove the core operational loop before expanding the product. Any feature that does not directly support check-in, status visibility, pickup, queue management, or pilot reporting should be deferred.
